# Digital ORRA - WhatsApp Business API Integration

A robust, production-ready WhatsApp API integration powered by **Google Gemini AI** and **Express.js**. This backend handles real-time webhooks, automatically replies to users with session-based memory, and provides custom messaging endpoints.

---

## 🚀 Transitioning from Development to Production (Publishing Your App)

When your app is in **Development Mode (Unpublished)**, Meta restricts webhooks: **only test webhooks** sent from the Meta App Dashboard will reach your server. No real-world production messages (even from admins or testers) will be delivered.

Follow these 5 steps to publish your app and start receiving real user messages:

### 1️⃣ Deploy the Server to a Public HTTPS URL
Meta webhooks require a secure `https://` endpoint. 

*   **Vercel Deployment (Recommended):**
    This project is pre-configured for Vercel. 
    1. Install the Vercel CLI: `npm install -g vercel`
    2. Deploy: `vercel`
    3. Configure your production Environment Variables on the Vercel Dashboard (see [Environment Variables](#environment-variables)).
*   **Alternative Hosting:** You can host it on Render, Heroku, AWS, or any provider of your choice, ensuring HTTPS is enabled.

### 2️⃣ Configure Basic App Settings on Meta Dashboard
Meta requires basic business information before an app can go live.
1. Go to the [Meta Developer Console](https://developers.facebook.com/).
2. Select your App.
3. In the left sidebar, navigate to **App Settings** ➡️ **Basic**.
4. Fill in the following mandatory fields:
   *   **Privacy Policy URL** (e.g., `https://yourdomain.com/privacy`)
   *   **Terms of Service URL** (e.g., `https://yourdomain.com/terms`)
   *   **Category** (e.g., *Business and Pages*)
5. Click **Save Changes** at the bottom.

### 3️⃣ Switch App Mode to "Live"
1. Locate the **App Mode** toggle in the top bar of your Meta App Dashboard.
2. Toggle it from **Development** to **Live**.
3. If prompted, complete the safety checks or specify the business portfolio.

### 4️⃣ Register a Real Phone Number
Test phone numbers cannot receive live WhatsApp messages from the public.
1. Go to **WhatsApp** ➡️ **API Setup** (or open the **WhatsApp Manager** via your Meta Business Suite).
2. Click **Add Phone Number**.
3. *Note: The number must not be currently linked to an active personal or business WhatsApp App on a phone. If it is, delete the WhatsApp account in your phone's app settings first.*
4. Follow the steps to verify the number via SMS or Voice Call.
5. In your Meta Business Manager, add a **Payment Method** to your WhatsApp Business Account (Meta offers a free tier of 1,000 service conversations per month, but a payment method is required to activate live messaging).

### 5️⃣ Update Webhook Callback & Verify Token
1. In your Meta App Dashboard, go to **WhatsApp** ➡️ **Configuration**.
2. Click **Edit** next to the Webhook Callback URL.
3. Set the **Callback URL** to: `https://your-deployed-domain.com/webhook`
4. Set the **Verify Token** to the value of `VERIFY_TOKEN` defined in your environment (e.g., `whatsapp_webhook_secret_verify_token_123`).
5. Click **Verify and Save**.
6. Under **Webhook Fields**, ensure you subscribe to **messages** and **message_deliveries**.

## 🛠️ Environment Variables

Create a `.env` file in the root directory (or add these to your hosting provider's dashboard):

```env
# Server Port
PORT=3000

# Meta API Credentials
WHATSAPP_TOKEN=your_production_whatsapp_access_token
PHONE_NUMBER_ID=your_production_phone_number_id
BUSINESS_ACCOUNT_ID=your_business_account_id
META_API_VERSION=v25.0

# Security (Choose a custom string to verify webhook authenticity)
VERIFY_TOKEN=your_secure_verify_token_here

# Gemini AI Integration (Get a free key from https://aistudio.google.com/)
GEMINI_API_KEY=your_google_gemini_api_key
SYSTEM_INSTRUCTION="You are a friendly, natural, and polite AI assistant from Digital ORRA..."
```

## 💻 Local Testing & Setup

If you want to test webhook behavior locally before deploying, follow these steps:

1. **Install dependencies:**
   ```bash
   npm install
   ```
2. **Start the server:**
   ```bash
   npm start
   ```
3. **Use Ngrok to tunnel traffic (For local testing):**
   ```bash
   ngrok http 3000
   ```
   *Copy the HTTPS URL generated by Ngrok and use it as your Callback URL on the Meta developer dashboard.*
4. **Run internal mocks to test response flow:**
   ```bash
   node test-webhook.js
   ```
5. **Send a test message manually via API:**
   ```bash
   node send-test-msg.js <phone_number_with_country_code> "Hello test message!"
   ```
